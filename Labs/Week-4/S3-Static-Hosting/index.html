What is Amazon S3?

Amazon S3, or Simple Storage Service, is AWS object storage. It is commonly used to store static files, backups, logs, images, videos, datasets, and website assets.

🪣

Bucket

A bucket is the container that stores objects. Bucket names must be globally unique.

📄

Object

An object is a file stored in S3, such as HTML, CSS, images, documents, or logs.

🔑

Key

A key is the unique name or path used to identify an object inside a bucket.

🕒

Versioning

Versioning keeps multiple versions of the same object and helps protect against deletion.

🌐

Static Website Hosting

S3 can serve static web content directly without managing an EC2 web server.

🛡️

Bucket Policy

A bucket policy controls who can access objects and what actions they can perform.

AWS Exam Tips

Remember these points for AWS Cloud Practitioner and Solutions Architect study.

  • S3 is object storage, not block storage and not a traditional file system.
  • Bucket names must be globally unique across AWS.
  • Objects are retrieved using their object key.
  • Static website hosting uses the S3 website endpoint.
  • Public websites need public read access to the website objects.
  • Versioning helps protect objects from accidental deletion or overwrite.
  • For private access to S3 from a VPC, use VPC endpoints.
  • For better performance, HTTPS, caching, and custom domains, pair S3 with CloudFront.
